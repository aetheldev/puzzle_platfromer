# Learning Method

Use this loop every time.

## Step 1 - Explain Concept

Keep it short.
2 to 4 sentences only.

## Step 2 - Read Specific Reference

Read only the specific proc or block in the solution.
Do not browse whole file.

## Step 3 - Answer One Question

Before writing code, answer one reading question.

## Step 4 - Close Reference

Close the solution.
Open blank `main.odin` in lesson folder.
Write from memory.

## Step 5 - Try First

Do not wait for perfect understanding.
Wrong code is useful.

## Step 6 - Review

Check:
- what is correct
- one thing to fix
- one question about your code

## Step 7 - Repeat

Only move on when:
- it compiles
- you can explain each important line
- you can change one thing without breaking it

## Rule About Solutions

- lesson folders under `learn/fundamentals/` are for your own code
- runnable answers live under `learn/solutions/fundamentals/`
- do not open the solution until the lesson tells you what exact lines to read
